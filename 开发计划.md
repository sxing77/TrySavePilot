# 海外华人返利省钱网站开发计划

版本：2026-05-01  
项目定位：给海外华人新手提供看得懂、能照着操作、不过度承诺的中文返利省钱指南。

## 1. 项目目标

本项目第一阶段不做大型返利平台数据库，而是做一个小而可信的内容型网站，帮助美国、加拿大、英国、澳洲等地区的中文用户理解 Rakuten、TopCashback、Honey 等返利 / 优惠工具的注册、购物、提现、不到账原因和使用边界。

第一阶段交付目标：

- 一个可上线的网站首页。
- 一个平台对比页面。
- 一个新手教程列表页。
- 一个购物场景列表页。
- 一个 FAQ 页面。
- 一个关于本站 / 合规披露页面。
- 首批 4-5 篇基础文章。
- 可维护的平台信息数据表。
- 基础访问与点击追踪方案。

## 2. MVP 范围

### 2.1 必做功能

- 首页展示网站定位、3 个新手入口、平台简表、最新文章、FAQ 摘要和推荐链接披露。
- 平台对比页展示 Rakuten、TopCashback、Honey 的核心差异。
- 教程文章页支持 Markdown / MDX 内容发布。
- 文章页自动展示最后更新时间、推荐链接披露和官方规则提醒。
- FAQ 页面集中解释返利不到账、提现、是否安全、优惠码冲突等问题。
- 关于本站页面说明本站非官方、可能包含推荐链接、联系方式和更新原则。
- 平台信息统一放在结构化数据文件中，避免页面里到处复制。
- 为推荐链接预留追踪参数或短链接字段。

### 2.2 暂不做功能

- 用户注册 / 登录。
- 返利账户绑定。
- 实时返利率抓取。
- 大规模品牌页面自动生成。
- 多语言站点。
- 评论系统。
- 后台 CMS。
- 自动检测推荐奖励变化。

## 3. 推荐技术方案

推荐使用静态内容站方案：

- 框架：Astro + TypeScript
- 样式：Tailwind CSS 或纯 CSS 模块
- 内容：Markdown / MDX
- 数据：本地 JSON / TypeScript 数据文件
- 部署：Cloudflare Pages、Vercel 或 Netlify
- 统计：Plausible、Google Analytics 或 Umami
- 搜索收录：Google Search Console

选择理由：

- 内容型网站对 SEO、速度、维护简单度要求高。
- 第一阶段没有复杂后端需求。
- Markdown 适合持续写教程文章。
- 静态部署成本低，适合先验证方向。

## 6. 开发阶段拆解

### 阶段 0：资料确认与项目初始化

预计时间：0.5-1 天

- 确认网站名称、域名方向和首发地区。
- 确认 3 个平台的官方页面、推荐链接和触发条件。
- 初始化项目结构。
- 配置代码规范、基础样式和页面路由。

交付物：

- 可运行的本地项目。
- 基础页面路由。
- 平台数据文件 v1。

### 阶段 1：首页与基础布局

预计时间：1-2 天

- 实现全站 Layout、Header、Footer。
- 实现首页首屏、新手入口、平台简表、文章入口和 FAQ 摘要。
- 添加全站推荐链接披露区域。
- 完成桌面端和移动端基础适配。

交付物：

- 首页 v1。
- 全站导航。
- 基础响应式布局。

### 阶段 2：平台对比与数据结构

预计时间：1 天

- 建立平台数据模型。
- 实现 `/compare` 对比页。
- 将推荐奖励、触发条件、提现方式、适合人群、不适合人群、更新时间统一维护。
- 添加“以官方页面为准”的规则提醒。

交付物：

- 平台对比页 v1。
- 平台信息数据文件 v1。

### 阶段 3：文章系统

预计时间：1-2 天

- 配置 Markdown / MDX 内容集合。
- 实现文章列表页。
- 实现文章详情页。
- 自动展示文章更新时间、分类、披露说明和官方规则提醒。
- 添加相关文章或返回列表入口。

交付物：

- `/guides` 页面。
- `/shopping` 页面。
- 文章详情模板。

### 阶段 4：首批内容录入

预计时间：2-4 天

- 写入 4 篇 P0 文章。
- 检查每篇文章是否包含披露、更新时间、触发条件和官方规则提醒。
- 避免“注册就送”“必到账”“稳赚”等表达。

交付物：

- 首批 4 篇基础文章。
- 合规检查记录。

### 阶段 5：FAQ、关于本站与基础 SEO

预计时间：1 天

- 完成 FAQ 页面。
- 完成关于本站页面。
- 添加每页 title、description、canonical。
- 添加 sitemap、robots.txt、Open Graph 信息。
- 配置基础结构化数据。

交付物：

- FAQ 页面。
- 关于本站页面。
- 基础 SEO 配置。

### 阶段 6：统计与上线

预计时间：0.5-1 天

- 接入访问统计。
- 为推荐链接点击预留事件追踪。
- 部署到正式环境。
- 提交 Google Search Console。
- 记录上线检查表。

交付物：

- 正式站点 v1。
- 数据追踪 v1。
- 上线检查结果。

## 8. 验收标准

### 8.1 功能验收

- 所有 P0 页面可以访问。
- 移动端和桌面端布局正常。
- 首页 3 个新手入口可点击。
- 平台对比表字段完整。
- 文章详情页显示更新时间和披露说明。
- 外链以新窗口打开，并有合理的安全属性。
- 推荐链接和官方链接可区分。

### 8.2 内容验收

- 每篇文章都有最后更新时间。
- 每篇文章都有推荐链接披露。
- 涉及奖励、返利、提现的内容均提醒以官方页面为准。
- 不出现“注册就送”“一定到账”“稳赚不赔”“官方推荐”等禁用表达。
- 不伪造收益、截图、用户评价。

### 8.3 SEO 验收

- 每个页面都有唯一 title 和 description。
- 生成 sitemap.xml。
- robots.txt 存在。
- 文章 URL 简洁可读。
- 首页和文章页首屏能明确说明网站主题。

### 8.4 数据验收

- 能记录页面访问。
- 能记录推荐链接点击。
- 能记录搜索展示和点击。
- 能手动记录收藏、私信、评论和用户问题。

## 9. 风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 平台推荐规则变化 | 内容过期或误导用户 | 每周检查官方页面，所有页面标注更新时间 |
| 内容像 AI 批量文 | 搜索表现差，信任低 | 写具体步骤、真实注意事项、FAQ 和场景判断 |
| 过度承诺收益 | 合规风险 | 使用固定披露模板和发布前检查清单 |
| 新站没流量 | 验证周期变长 | 从“不到账”“安全吗”“怎么提现”等问题型关键词切入 |
| 推荐链接转化低 | 商业闭环弱 | 优化教程细节、透明说明和平台选择建议 |
