# 海外华人返利省钱网站详细设计文档

版本：2026-05-01  
状态：MVP 设计稿  
适用范围：首页、平台对比、文章系统、FAQ、合规披露、数据追踪。

## 1. 产品定位

本网站是一个面向海外华人新手的中文返利省钱指南。网站通过教程、对比表和 FAQ 解释返利平台怎么注册、怎么购物、怎么提现、为什么不到账，以及不同平台适合哪些用户。

网站不冒充官方，不承诺收益，不提供实时最高返利率。所有涉及奖励、提现、返利比例、活动条件的信息都需要标注更新时间，并提醒用户以平台官方页面为准。

## 2. 用户画像

### 2.1 主要用户

- 刚到美国、加拿大、英国、澳洲的留学生。
- 新移民和刚开始海外网购的中文用户。
- 经常海淘但不了解 cashback 的用户。
- 想比较 Rakuten、TopCashback、Honey 的用户。

### 2.2 用户核心问题

- 返利平台是不是安全？
- 注册后怎么购物才算有效？
- 奖励是不是注册就有？
- 为什么返利没有到账？
- 怎么提现？
- 优惠码、礼品卡、信用卡返现会不会影响返利？
- Rakuten 和 TopCashback 哪个更适合新手？

## 4. 页面设计

### 4.1 首页 `/`

目标：让用户 5 秒内看懂网站做什么，并进入最适合自己的内容。

页面模块：

1. 顶部导航
   - 首页
   - 平台对比
   - 新手教程
   - 购物场景
   - 常见问题
   - 关于本站

2. 首屏
   - 主标题：海外华人返利省钱指南
   - 副标题：用中文看懂 Rakuten、TopCashback、Honey 等返利平台怎么用。
   - 主按钮：查看 Rakuten 新手教程
   - 次按钮：查看 TopCashback 教程
   - 次按钮：打开返利平台对比表
   - 透明说明：本站可能包含推荐链接，规则以官方页面为准。

3. 适合人群 / 不适合人群
   - 适合：海外网购新手、想比较平台、愿意按步骤操作的人。
   - 不适合：期待立即到账、不愿确认规则、不常网购的人。

4. 平台简表
   - Rakuten
   - TopCashback
   - Honey / PayPal Honey

5. 新手阅读顺序
   - 先看平台是什么。
   - 再看注册和购物步骤。
   - 下单前确认触发条件。
   - 返利未到账时查看 FAQ。

6. 最新文章
   - 展示 4-6 篇。

7. FAQ 摘要
   - 返利平台可靠吗？
   - 为什么没有到账？
   - 推荐奖励是不是注册就有？
   - 返利可以提现吗？

8. 页脚披露
   - 推荐链接披露。
   - 非官方说明。
   - 更新时间说明。

### 4.2 平台对比页 `/compare`

目标：帮助用户快速判断哪个平台适合自己。

核心字段：

| 字段 | 说明 |
| --- | --- |
| 平台名称 | Rakuten / TopCashback / Honey |
| 平台定位 | 返利平台、优惠券插件、积分工具等 |
| 适合地区 | 美国、加拿大、英国等 |
| 主要用途 | cashback、找优惠券、比价等 |
| 注册门槛 | 是否免费、是否需要邮箱 |
| 推荐奖励 | 当前活动说明，必须标注更新时间 |
| 触发条件 | 合格消费金额、时间窗口、订单确认 |
| 提现方式 | PayPal、支票、银行等 |
| 到账周期 | 订单确认、月结、季度等 |
| 适合人群 | 新手、高频购物者、只想找券的人 |
| 不适合人群 | 不愿等待确认、不常购物的人 |
| 官方规则链接 | 用于核对最新规则 |

页面结论不直接强推单个平台，而是按场景推荐：

- 第一次使用返利：优先选择流程最容易理解的平台。
- 高频网购：同时比较返利率和到账稳定性。
- 只想自动找优惠券：考虑 Honey。
- 对提现方式敏感：优先看提现方式和门槛。

### 4.3 新手教程列表 `/guides`

目标：集中承接平台名 + 怎么用、注册、提现、对比等搜索需求。

列表字段：

- 文章标题
- 摘要
- 平台
- 阅读时间
- 最后更新时间
- 标签

首批文章：

- 美国 Rakuten 怎么用？新手注册、购物、提现完整教程
- TopCashback 怎么用？和 Rakuten 有什么区别？
- Rakuten vs TopCashback：哪个更适合新手？
- 海外华人常用返利平台对比表：提现方式、门槛、适合人群

### 4.4 文章详情页 `/guides/[slug]`

目标：提供可信、可照着操作的长文教程。

固定结构：

1. 标题
2. 摘要
3. 最后更新时间
4. 推荐链接披露
5. 官方规则提醒
6. 正文目录
7. 正文内容
8. FAQ
9. 相关文章

文章正文要求：

- 说明平台是什么。
- 说明适合谁、不适合谁。
- 分步骤写注册流程。
- 分步骤写购物返利流程。
- 明确奖励和返利通常需要满足条件。
- 写清楚提现方式和门槛。
- 写常见不到账原因。
- 结尾重复提醒以官方规则为准。

### 4.5 购物场景页 `/shopping`

目标：承接 Nike、iHerb、Sephora、Booking 等具体购物搜索。

文章类型：

- 在 XX 买东西怎么拿返利？
- XX 优惠码、返利、信用卡叠加思路
- XX 返利不到账常见原因

每篇场景文章必须提醒：

- 返利率会变化。
- 优惠码、礼品卡、退货、广告拦截可能影响返利。
- 下单前需要重新确认平台规则。

### 4.6 FAQ 页面 `/faq`

问题分组：

1. 基础问题
   - 返利网是什么？
   - 返利平台为什么会给钱？
   - 返利会不会让商品变贵？

2. 到账问题
   - 为什么用了返利网却没有到账？
   - 多久能到账？
   - 退货会影响返利吗？
   - 用优惠码会影响返利吗？

3. 提现问题
   - 可以提现吗？
   - 有最低提现门槛吗？
   - 支持 PayPal 吗？

4. 合规与安全
   - 本站是不是官方？
   - 什么是推荐链接？
   - 本站是否会获得奖励？

### 4.7 关于本站 `/about`

必须包含：

- 本站是个人整理的中文指南，不代表任何平台官方。
- 本站可能包含 referral / affiliate 链接。
- 用户通过链接注册或消费后，本站可能获得奖励，但不会增加用户成本。
- 平台规则、返利比例、奖励条件、提现规则可能变化。
- 页面内容以官方最新说明为准。
- 联系方式。

## 7. 合规设计

### 7.1 固定披露文案

标准版：

> 本文可能包含推荐链接。你通过这些链接注册或消费后，本站可能获得平台奖励，但这不会增加你的成本。返利比例、奖励条件和提现规则可能变化，请以平台官方页面为准。

短版：

> 说明：本文含推荐链接，我可能获得平台奖励；规则以官方页面为准。

强提醒版：

> 注意：部分平台奖励不是“注册即到账”，通常需要满足合格消费、时间窗口或 cashback 确认等条件。

### 7.2 禁用表达

| 禁用表达 | 推荐替换 |
| --- | --- |
| 注册就送 $50 | 满足官方条件后，可能获得 $50 奖励 |
| 必到账 | 通常需要订单确认，具体以平台规则为准 |
| 稳赚不赔 | 这是一个省钱工具，不是赚钱保证 |
| 最高返利一定在 XX | 不同时间返利率会变化，建议下单前再确认 |
| 官方推荐 | 本文为个人整理，不代表平台官方 |

### 7.3 发布前检查

每篇文章发布前必须检查：

- 是否写清楚推荐链接披露。
- 是否标注最后更新时间。
- 是否避免绝对化承诺。
- 是否写清楚奖励触发条件。
- 是否提醒以官方页面为准。
- 是否没有伪造截图、收益和用户评价。
- 是否没有冒充平台官方。
- 是否没有用模板批量替换品牌名生成低价值页面。

## 8. SEO 设计

### 8.1 URL 规则

- 使用英文 slug。
- 保持简短、稳定。
- 不在 URL 中写活动金额或短期奖励。

示例：

- `/guides/rakuten-guide`
- `/guides/topcashback-guide`
- `/guides/rakuten-vs-topcashback`
- `/shopping/nike-cashback`

### 8.2 Meta 规则

每页必须有：

- `title`
- `description`
- `canonical`
- Open Graph title
- Open Graph description

文章页 title 示例：

```text
美国 Rakuten 怎么用？新手注册、购物、提现完整教程
```

description 示例：

```text
用中文了解 Rakuten 是什么、怎么注册、怎么通过商家购物拿返利、奖励触发条件、提现方式和常见不到账原因。规则可能变化，请以官方页面为准。
```

### 8.3 结构化数据

可添加：

- `Article`
- `FAQPage`
- `BreadcrumbList`

### 8.4 内容质量原则

- 每篇文章必须解决一个具体问题。
- 不批量生成只有品牌名不同的页面。
- 表格、步骤、FAQ 要提供真实判断价值。
- 涉及数字的地方标注更新时间和官方来源。

## 9. 数据追踪设计

### 9.1 需要追踪的指标

| 指标 | 工具 | 用途 |
| --- | --- | --- |
| 页面访问 | GA / Plausible / Umami | 判断是否有人看 |
| 搜索展示 | Google Search Console | 判断搜索引擎是否理解页面 |
| 推荐链接点击 | 事件追踪 / 短链接 | 判断是否有转化意图 |
| 收藏 / 私信 / 评论 | 手动表格 | 判断内容是否有帮助 |
| 转化奖励 | 平台后台 | 判断商业闭环是否成立 |

### 9.2 推荐链接点击事件

事件字段：

```ts
type AffiliateClickEvent = {
  platformId: string;
  placement: "home" | "compare" | "article" | "footer";
  articleSlug?: string;
  linkType: "referral" | "official";
};
```

### 9.3 30 天判断规则

| 情况 | 判断 | 下一步 |
| --- | --- | --- |
| 有访问、有收藏、无转化 | 方向可能对，信任和转化还不够 | 加强教程细节、真实截图、FAQ |
| 有访问、无收藏、无点击 | 标题有吸引力，内容价值不足 | 重写结构，增加表格和步骤 |
| 无访问 | 渠道或关键词不对 | 从不到账、安全、提现等问题切入 |
| 有推荐链接点击 | 用户开始信任入口 | 优化该文章，扩展同类选题 |
| 有转化奖励 | 商业闭环成立 | 扩展到 5-8 个平台或更多场景 |

## 10. 视觉与交互原则

整体风格：清楚、可信、工具型、适合中文新手阅读。

设计要求：

- 首屏直接说明网站价值，不做营销式夸张表达。
- 表格要易扫读，移动端可横向滚动或转为卡片。
- 推荐链接按钮旁边必须有透明说明。
- 文章正文宽度控制，优先阅读体验。
- FAQ 支持展开收起。
- CTA 文案避免诱导式表达，使用“查看教程”“打开官方页面”“查看对比表”等中性文字。
